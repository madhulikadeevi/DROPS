The existing cloud storage system relies on traditional Iris file systems and standard cryptographic techniques like AES (Advanced Encryption Standard) and DES (Data Encryption Standard) for securing data. These techniques focus on data integrity (ensuring no modifications), confidentiality (keeping it private), and availability (ensuring access when needed).

However, this system allows users to choose their own encryption methods, which can lead to inconsistent security levels. Some users might select weaker security settings, creating vulnerabilities.

How the Existing System Works:

  1. Data Storage: Users upload their files to the cloud.
  2. Encryption: The data is encrypted using AES or DES.
  3. Storage in Cloud: The encrypted file is stored in the cloud, relying on the Iris File System to manage data placement.
  4. User Access: When needed, the user decrypts the file using their encryption key.

2. Problems and Drawbacks in the Existing System

A. Security Vulnerabilities Due to User-Selected Confidentiality

  • Users choose encryption schemes, meaning some may opt for weaker security settings.
  • This inconsistency creates security gaps, allowing attackers to target weakly encrypted data.
  • Some encryption techniques may not be updated regularly, exposing data to new cyber threats.

B. Performance Issues with Traditional Cryptography

  • AES and DES encryption are computationally expensive, meaning they consume a lot of resources (CPU power and memory).
  • When dealing with large-scale cloud storage, encryption and decryption become slow and costly.
  • This leads to performance degradation, making cloud operations less efficient.

C. Risk of Critical Data Leakage

  • If encryption keys are compromised, hackers can decrypt the data.
  • Improper sanitization (when old data is not completely removed) can leave traces of sensitive information.
  • Malicious Virtual Machines (VMs) can try to gain unauthorized access to stored data. If a VM gets infected, the entire cloud storage might be at risk.

D. Data Integrity Issues and Tampering Risks

  • The existing system does not have a strong way to prevent data tampering.
  • Attackers may modify encrypted data without being detected.
  • This could lead to corrupted or incorrect data being retrieved, affecting reliability.

3. Algorithms Used in the Existing System

The traditional system mainly relies on two encryption algorithms for security:

A. Advanced Encryption Standard (AES)

  • AES is a widely used symmetric encryption algorithm, meaning the same key is used for encryption and decryption.
  • It works by converting plaintext into ciphertext using 128-bit, 192-bit, or 256-bit keys.
  • Strengths:
    • Highly secure and used worldwide.
    • Efficient for small-scale data encryption.
  • Weaknesses:
    • Slow when dealing with large datasets in cloud storage.
    • If the encryption key is leaked, attackers can decrypt all data.

B. Data Encryption Standard (DES)

  • DES is an older encryption method using 56-bit keys to encrypt and decrypt data.
  • Strengths:
    • Used for secure communication in older systems.
  • Weaknesses:
    • Weak security modern computers can break DES encryption within hours.
    • Not recommended for cloud storage due to its vulnerability to brute-force attacks.

C. Iris File System

  • The Iris File System is a distributed storage system used to manage data across cloud servers.
  • It organizes files and controls who can access them.
  • However, it does not provide strong security against VM escape attacks or improper sanitization issues.

How DROPS Solves the Problems in the Existing System

Instead of relying on encryption-heavy methods that slow down performance, DROPS enhances security by breaking data into fragments and distributing them across multiple cloud servers.

  1. Data Fragmentation:

    • A file is split into smaller pieces (fragments) before storing it in the cloud.
    • No single server has the complete file, making it impossible for hackers to reconstruct it.

  2. Smart Distribution:

    • The fragments are distributed across different cloud nodes, reducing security risks.
    • This prevents VM escape attacks, where attackers compromise a virtual machine to steal full files.

  3. Selective Replication:

    • Some fragments are replicated across multiple servers to ensure availability.
    • If one server fails, the data is still accessible from another server.

  4. Centrality-Based Placement:

    • Instead of placing data randomly, DROPS uses centrality measures to intelligently decide where to store each fragment.
    • This balances security and fast access, improving cloud performance.
